# 用户态工具

## libselinux

核心作用：这是SELinux的基础共享库，提供了应用程序与内核SELinux模块交互的API接口。
功能详解：

它包含了一系列底层函数（如 selinux_check_access, getenforce, setenforce），允许其他程序查询安全上下文、检查访问权限、获取或设置 enforcing 模式。

大多数其他SELinux用户态工具（如下面的policycoreutils）都依赖于此库来执行实际操作。
包含用于处理文件上下文映射（labeling）的工具，如 matchpathcon 和 restorecon 的底层逻辑支持。

## policycoreutils

核心作用：这是SELinux策略管理的核心工具集，包含了管理系统策略、文件标签和布尔值的最常用命令行工具。

功能详解：
策略管理：semodule 用于加载、安装、升级和移除SELinux策略模块；load_policy 用于重新加载整个策略。
状态查看：sestatus 用于查看当前SELinux的状态（是否启用、模式、加载的策略版本等）。
文件标签管理：restorecon 和 setfiles 用于根据策略定义恢复或设置文件的正确安全上下文（Label）。
布尔值管理：setsebool 和 getsebool 用于永久或临时修改SELinux布尔值开关，以调整策略行为而不重写策略。
角色切换：newrole 允许用户在不同的SELinux角色之间切换。

## policycoreutils-python-utils

核心作用：提供基于Python的高级SELinux管理工具，主要用于策略分析和自动化审计规则生成。

功能详解：
审计分析：audit2allow 是最常用的工具之一，它分析审计日志（AVC denials），自动生成允许这些被拒绝操作的策略模块代码。
原因解释：audit2why 解释为什么特定的访问会被拒绝，并提供解决建议。
高级管理：semanage 是一个强大的命令行工具，用于管理SELinux策略的各个组成部分（如端口定义、文件上下文映射、用户映射、布尔值等），而无需直接编译策略源码。
类别管理：chcat 用于更改文件的类别集（Category Set）。

## setools-console

核心作用：提供一套命令行工具，用于深入分析、查询和验证SELinux策略数据库。

功能详解：
策略查询：sesearch 允许用户搜索策略中的特定规则（如谁可以访问什么类型的文件）；seinfo 统计并显示策略组件（类型、角色、用户等）的数量和详细信息。
策略比较：sediff 用于比较两个不同策略文件之间的差异。
策略检查：sechecker 用于运行一系列测试以验证策略的完整性和一致性。
这些工具对于策略开发者和高级管理员排查复杂的权限问题至关重要。

## setroubleshoot-server

核心作用：提供SELinux故障诊断和通知服务，旨在简化普通用户和管理员对SELinux拦截事件的理解。

功能详解：
自动分析：守护进程 setroubleshootd 监控审计日志，当发现AVC拒绝记录时，自动分析原因。
生成报告：sealert 工具会生成人类可读的错误报告，解释发生了什么、为什么被阻止，并给出具体的修复命令（通常是运行 restorecon 或 setsebool 的命令）。
通知机制：在桌面环境中，它通常会弹出通知窗口提醒用户有安全拦截发生。

## policycoreutils-devel

核心作用：包含用于开发和生成自定义SELinux策略的开发工具和辅助脚本。

功能详解：
策略生成：sepolicy 工具可以帮助生成新策略模块的模板代码，或者基于现有可执行文件生成策略骨架。
接口生成：sepolgen 系列工具用于从源代码或二进制文件中提取接口信息，辅助策略编写。
主要面向策略开发者，用于创建新的 .te (Type Enforcement) 文件。

## mcstrans

核心作用：提供MLS（多级安全）和MCS（多类别安全）上下文的转换和显示工具。

功能详解：
上下文翻译：将内部复杂的数字或代码形式的敏感级别（如 "s0:c1,c2"）转换为人类可读的名称（如果配置了翻译表）。
颜色显示：mlscolor-test 等工具用于在终端中用颜色区分不同的安全级别，提高可读性。
主要在启用了MLS/MCS功能的严格安全环境中使用。

## audit

核心作用：Linux审计框架的用户态工具集，虽然不仅限于SELinux，但它是SELinux记录拒绝日志（AVC Denials）的基础设施。

功能详解：
规则管理：auditctl 用于配置内核审计系统的规则（例如监控特定文件或系统调用）。
日志查询：ausearch 和 aureport 用于搜索审计日志文件并生成报告。SELinux的拒绝记录通常存储在 /var/log/audit/audit.log 中。
守护进程：auditd 是负责将内核审计事件写入磁盘日志的守护进程。没有它，SELinux的拒绝记录将无法持久化保存，导致 audit2allow 等工具无法工作。

## 总结

* 基础库：libselinux
* 日志记录：audit (记录问题) -> setroubleshoot-server (分析问题并报警)
* 日常管理：policycoreutils (改状态、改标签、加载策略) + policycoreutils-python-utils (智能修复、高级配置)
* 深度分析与开发：setools-console (查策略细节) + policycoreutils-devel (写新策略)
* 特殊环境：mcstrans (MLS/MCS支持)
